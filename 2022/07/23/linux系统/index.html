linux系统学习笔记

发表于 2022-07-23 分类于 操作系统 ， linux 阅读次数： Valine：
本文字数： 1.1k 阅读时长 ≈ 1 分钟

基础知识

更改shell

使用chsh命令可以更改shell（如改为ksh或者tcsh）

环境变量和shell变量

shell中可以保存一些临时变量，称为shell变量，可以用来保存脚本执行过程中的数据

STUFF=blah

这一语句将blah赋值给shell变量STUFF

可以用echo $STUFF 来显示变量STUFF的值

环境变量 shell变量
能够被shell中运行的所有进程访问 只能被当前的shell访问，在shell中运行的命令则无法访问

命令路径

echo $PATH

可以看到所有的路径组件，彼此以：分隔

PATH=dir:$PATH PATH=$PATH1:dir
优先查找dir路径 最后查找dir路径

shell输入输出

command > file

将命令的执行结果输出到文件（默认是屏幕），如果文件不存在，将会被创建，如果存在，将会清空其中的内容

如果不想覆盖文件，可以加入到文件末尾

command >> file

可以使用管道字符|将一个命令的执行结果输出到另一个命令

head /proc/cpuinfo | tr a-z A-Z

标准输入重定向

head < /proc/cpuinfo

将文件内容重定向为命令的标准输入

查看和操纵进程

可以用ps命令列出所有正在运行的进程

  1. PID 进程ID
  2. TTY 进程所在的终端设备
  3. STAT 进程状态，S休眠，R运行 可参考 man 1 ps
  4. TIME 进程目前为止所用CPU时长，就是进程占用CPU的总时长
  5. COMMAND 命令名

一些实用的选项组合

  1. ps x 显示当前用户运行的所有进程
  2. ps ax 显示系统当前运行的所有进程，包括其他用户的进程
  3. ps u 显示更详细的进程信息
  4. ps w 显示命令的全名，而非仅显示一行以内的内容

注意可以把PID当成ps命令的一个参数，用来查看该特定进程的信息

ps u 是一个shell变量，表示当前的shell进程

终止进程

kill pid

暂停进程

kill -STOP pid

文件模式和权限

当我们使用ls -l 来查看文件信息后

-rw-r—r— 1 juster somegroup 7041 Mar 26 19:34 endnotes.html

其中第一个-，指的是文件类型。-代表常规文件，d代表目录

后面的九个字符代表文件权限信息，平均分成三部分，用户.用户组.其他

r 文件可读

w 文件可写

x 文件可执行

更改文件权限

chmod g+r file

chmod o+r file

这样就给用户组g和其他用户o加上了可读权限r

也可以 chmod go+r file

如果想要取消权限，可以go-r

也可以使用绝对权限设置，使用数组来代表权限

chmod 644 file

归档和压缩文件

压缩文件 gzip file

解压缩文件 gunzip file.gz

gzip命令只可以压缩单个文件
